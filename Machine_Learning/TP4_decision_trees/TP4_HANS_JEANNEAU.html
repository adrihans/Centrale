1.

The cell above creates a decision tree model and trains it on the train set.

The above representation of the tree works the following way:

  • Each non-leaf node represents a linear boundary on one variable: feature < value
  • Each node represents a point in the process, with additional informations: Gini coefficient, number of sample, chosen class
  • Each leaf node represents the end of the classification
  • The color of the node represents the classification: orange for setosa, green for versicolor, purple for veronica, and in-betweens when the node is not pure.

2.

The dataset is in 4 dimensions, it is therefore not trivial to visualize it. To do so, we want to project our dataset on a plane, plot the data and the boundary. We chose to project the data on the canonical base , i.e on two chosen features. We managed to do so using the numpy function contourf on a meshgrid of the two chosen features, concatenated with default values for the discarded features.

In [87]:
def frontieres(clf, train, i1, i2, j1, j2):
    # j1, j2 are the features selected
    # i1, i2 are the features discarded
    X = train[[features[j1], features[j2]]].to_numpy()
    # We take the average value of each discarded value as default value
    mean1 = (train[features[i1]].min() + train[features[i1]].max())/2
    mean2 = (train[features[i2]].min() + train[features[i2]].max())/2
    
    h = .02
    # We create a grid of the values to display ...
    x1_min, x1_max = X[:, 0].min() - .5, X[:, 0].max() + .5
    x2_min, x2_max = X[:, 1].min() - .5, X[:, 1].max() + .5
    x11, x22 = np.meshgrid(np.arange(x1_min, x1_max, h),np.arange(x2_min, x2_max, h))
    # Create arrays of by-default values for the discarded features ...
    ln = product(x11.shape)   
    t1, t2 = np.array([mean1]*ln).reshape(-1, 1), np.array([mean2]*ln).reshape(-1, 1)
    # And concatenate them at the right place.
    to_concat = [0, 0, 0, 0]
    to_concat[i1], to_concat[i2] = t1, t2
    to_concat[j1], to_concat[j2] = x11.ravel(), x22.ravel()
    A = np.column_stack(to_concat)
    # Prediction time
    Z = clf.predict(A)
    Z=Z.reshape(x11.shape)

    plt.contourf(x11, x22, Z, levels=[-0.1, 0.9, 1.9, 2.1], cmap=plt.cm.bwr, alpha=.8)

def trainmodel(sentence, test_species=test['species'],
               train=train[features], test=test[features],
               y_train=y_train, y_test=y_test, **kwargs):
    # Create and fit the model
    clf = tree.DecisionTreeClassifier(**kwargs)

Feature selection using random forests byproducts¶

One of the interesting use cases for random forest is feature selection. One of the byproducts of trying lots of decision tree variations is that you can examine which variables are working best/worst in each tree.

When a certain tree uses one variable and another doesn't, you can compare the value lost or gained from the inclusion/exclusion of that variable. The good random forest implementations are going to do that for you, so all you need to do is know which method or variable to look at.

View feature importance¶

While we don't get regression coefficients like with ordinary least squares (OLS), we do get a score telling us how important each feature was in classifying. This is one of the most powerful parts of random forests, because we can clearly see that petal width was more important in classification than sepal width.

In [60]:
def trainRF(sentence, N=20, test_species=test['species'],
            train=train[features], test=test[features],
            y_train=y_train, y_test=y_test, **kwargs):
    start = time.time()
    avg_score = 0
    # training of N random forests
    for n in range(N):
        clf = RandomForestClassifier(**kwargs)
        clf = clf.fit(train, y_train)
        # Update of the score
        avg_score += 1/N * clf.score(test, y_test)
    end = time.time()
    # Print the average score
    print(sentence.format(avg_score))
    dot_data = tree.export_graphviz(clf.estimators_[0], out_file=None, 
                         feature_names=iris.feature_names,  
                         class_names=iris.target_names,  
                         filled=True, rounded=True,  
                         special_characters=True)
    # Display one tree
    print('one tree:')
    display(graphviz.Source(dot_data))
    # print the average execution time
    print('average time: {:.0f} ms'.format((end-start)/N*1e3))

finally, the max_leaf_nodes prameter controls the maximum number of leaves nodes in each tree. If reduced too much, the trees won't be able to make enough splits to get correct performances, and the forest will underfit.

Above all the parameters above, many have an impact on the tendency of the model to overfit, but on different ways: tree depth, number of leaves, number of sample per leave, etc. A good model should combinate the above parameters to find the optimal spot. To do so, we chose to use a gridsearch that will explore exhaustively combinations of the parameters. We chose revelant values for each parameters using our understanding of their effects, and let the gridsearch give us the combination leading to the best performances.

4.

Random Forests are Bagging models: they consists of a bootstrapping of the data (sampling the dataset according to its own distribution) for each estimator (here, decision trees) and aggregating the results (on a majority vote).

The bootstraping step implies that for every decision tree, some data points are not included in its training set and could be reused for estimating the performances of the model. The data points excluded from the training set are called out-of-sample points.

The out-of-bag score is computed as following: For each point which is not included in at least one estimator, find all models that are not including it in its training set, evaluate it, and make a majority vote beetween these estimators. Comparing all these results to the ground truths gives the OOB score.

This score is especially useful when we do not have enough data to afford to isolate some of it for a training set. Also, we can prove that when the dataset is big enough, the out-of-bag sample consists in 32% of the total dataset, making it a reasonable size for model evaluation. The probability of not picking N rows in random draws using sampling-with-replacement is $(\frac{N-1}{N})^N = (1 + \frac{1}{N})^N$, which tends towards $e^{-1} \simeq 0.3168$ when N tends toward infinity

5.

The parameters class_weight allows to modify the weight given to different classes, and balance unbalanced dataset. The best option when we don't have enough information on the data is clas_weight = balanced, which weight each class inversely proportionally their apparition frequency in the dataset: $w_j=\frac{N}{k*n_j}$ with $w_j$ the weigth of the class j, $N$ the number of data points, $n_j$ the number of data points in the class j and $k$ the number of classes.

def prune(decisiontree, min_samples_leaf = 1):
    if decisiontree.min_samples_leaf >= min_samples_leaf:
        raise Exception('Tree already more pruned')
    else:
        decisiontree.min_samples_leaf = min_samples_leaf
        tree = decisiontree.tree_
        for i in range(tree.node_count):
            n_samples = tree.n_node_samples[i]
            if n_samples <= min_samples_leaf:
                tree.children_left[i]=-1
